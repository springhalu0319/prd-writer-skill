# PRD 质量检查清单

完成PRD后，以四个角色视角进行检查：

## 1. 技术负责人视角 🔧

从技术实现难度、系统性能、安全风险角度审视：

### 检查问题
- [ ] 并发处理：高峰期如何保证系统稳定？
- [ ] 幂等性：重复提交如何处理？
- [ ] 异常恢复：网络断开、支付中断如何处理？
- [ ] 数据一致性：分布式场景下如何保证？
- [ ] 安全防护：SQL注入、XSS、CSRF如何防范？
- [ ] 性能瓶颈：大数据量查询如何优化？

### 示例问题
> "如果用户在支付过程中网络断开，订单状态如何处理？是否有幂等性设计？"

---

## 2. 挑剔用户视角 👤

从操作便捷性、流程合理性角度审视：

### 检查问题
- [ ] 操作步骤是否过多？能否减少点击次数？
- [ ] 表单字段是否过多？能否智能填充？
- [ ] 错误提示是否友好？用户能否理解？
- [ ] 加载状态是否清晰？用户是否知道在等待？
- [ ] 撤销/回退是否方便？

### 示例问题
> "为什么修改收货地址要跳转三个页面？能不能在订单确认页直接编辑？"

---

## 3. 运营负责人视角 📊

从数据分析、营销推广角度审视：

### 检查问题
- [ ] 埋点设计：关键行为是否有埋点？
- [ ] 转化追踪：营销活动效果如何评估？
- [ ] 用户分层：如何区分不同价值用户？
- [ ] 运营工具：是否有足够的后台配置能力？
- [ ] 数据导出：运营报表如何生成？

### 示例问题
> "是否有用户行为埋点？如何追踪营销活动的转化效果？"

---

## 4. 测试工程师视角 🧪

从异常场景覆盖角度审视：

### 检查问题
- [ ] 边界值：最大/最小/空值如何处理？
- [ ] 并发操作：多人同时操作同一数据？
- [ ] 网络异常：弱网、断网、超时？
- [ ] 设备兼容：不同屏幕、浏览器、系统？
- [ ] 权限边界：越权操作是否被拦截？

### 示例问题
> "如果用户重复点击提交按钮会怎样？是否有防重复提交机制？"

---

## 检查输出格式

完成检查后，输出补充建议：

```markdown
## 质量检查报告

### 技术风险
1. [问题描述] → 建议：[解决方案]

### 用户体验
1. [问题描述] → 建议：[解决方案]

### 运营需求
1. [补充需求] → 建议：[实现方案]

### 测试要点
1. [测试场景] → 预期：[预期结果]
```
